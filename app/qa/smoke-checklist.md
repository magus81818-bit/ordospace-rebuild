# 화면 점검 목록

구조를 나눈 뒤 매번 확인할 최소 점검 목록입니다.

## 공개 화면

- `/` 또는 `index.html`: 랜딩 화면이 보입니다.
- `/#auth`: 로그인 화면이 보입니다.
- `/#inquiry`: 도입 문의 모달이 보입니다.
- `/#terms`: 이용약관 화면이 보입니다.
- `/#privacy`: 개인정보처리방침 화면이 보입니다.
- `/#support`: 고객지원 화면이 보입니다.

## 역할별 화면

- client 홈 화면이 깨지지 않습니다.
- admin 홈 화면이 깨지지 않습니다.
- worker 홈 화면이 깨지지 않습니다.

## 공통 동작

- 모바일 드로어가 열리고 닫힙니다.
- 사이드 메뉴가 역할별로 바뀝니다.
- Lucide 아이콘이 보입니다.
- 주요 버튼 클릭 후 콘솔 오류가 나지 않습니다.

## 구조 검증 원칙

한 번에 많이 옮기지 않습니다. 새 영역으로 이동한 뒤에는 이 점검 목록을 다시 확인합니다.

## 자동 점검 명령

구조를 바꾼 뒤에는 수동 확인 전에 아래 명령을 먼저 실행합니다.

```powershell
node app/qa/smoke-check.js
```

배포된 Vercel 화면까지 확인할 때는 아래 명령을 사용합니다.

```powershell
node app/qa/smoke-check.js --url https://ordospace-sprint5.vercel.app/
```

이 자동 점검은 랜딩, 로그인, client 주요 화면, admin 홈, worker 홈을 실제 브라우저에서 열어 화면 표시와 주요 렌더 영역을 확인합니다. favicon 404는 실패로 보지 않습니다.
