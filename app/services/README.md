# services

서비스 파일은 특정 시각 컴포넌트에 묶이지 않는 업무 흐름 규칙을 담당합니다.

## 현재 파일

- `session.service.js`
  - 현재 역할을 `localStorage`에 저장합니다.
  - 활성 역할을 `window.ORDO_ROLE`로 노출합니다.
  - 역할이 바뀌면 알맞은 홈 화면으로 이동시킵니다.
  - 로그인 이메일에서 데모 역할을 판단합니다.
  - 가짜 세션 객체를 `window._ordoSession`에 만듭니다.

## 향후 후보

- 승인, 반려, 보류 같은 승인 상태 변경.
- 도입 문의 제출 처리.
- client/admin/worker 화면이 함께 쓰는 프로젝트 액션 규칙.

화면은 버튼과 폼을 담당하고, 서비스는 그 액션 뒤의 규칙을 담당합니다.
