# router

화면 이동을 담당하는 영역입니다.

## 현재 파일

- `hash-router.js`
  - `#landing`, `#auth`, `#dashboard`, `#inquiry`처럼 URL hash를 읽고 어떤 화면을 보여줄지 결정합니다.

## 담당 범위

- hash 기반 화면 전환.
- 경로 별칭 정리.
- 권한 없는 화면 접근 시 403 화면으로 보내는 보호 규칙.
- 랜딩 페이지 내부 섹션 스크롤 이동.
- 사이드바, 드로어, 모바일 탭의 활성 상태 동기화.

## 담당하지 않는 범위

- 로그인 여부 판단은 `app/services/`가 담당합니다.
- 실제 데이터 조회는 `app/repos/`가 담당합니다.
- 화면 HTML 조립은 `app/screens/`와 `app/ui/`가 담당합니다.
