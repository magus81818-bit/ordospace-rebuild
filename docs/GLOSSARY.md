# GLOSSARY — 용어집

> 코드와 가이드에 나오는 용어를 비개발자 기준으로 풀어씁니다.
> 각 주석화 라운드에서 새 용어가 나올 때마다 여기에 추가됩니다. (가나다 → ABC 순)

## 한국어 용어

| 용어 | 뜻 |
|---|---|
| **부수효과** | 함수가 "값을 돌려주는 것" 말고 몰래 하는 일. 예: 저장하기, 화면 다시 그리기, 알림 보내기. 부수효과를 모르면 "이 함수 불렀는데 왜 저장이 됐지?"가 됨 |
| **상태(status)** | 카드가 지금 어느 단계에 있는지를 나타내는 꼬리표. `pending`(대기) → `in_progress`(진행중) → `review`(PM 검토중) → `done`(클라이언트 승인 대기) → `approved`(승인완료). 옆길로 `revision`(수정요청) |
| **시드 데이터(seed)** | 시연을 위해 미리 심어둔 샘플 데이터. `app/data/workspace.data.js`의 카드 12장이 그것 |
| **역할(role)** | admin(관리자/PM), worker(작업자), client(클라이언트). 역할에 따라 보이는 화면과 누를 수 있는 버튼이 다름 |
| **이정표** | 각 파일 헤더의 `[다음 읽을 파일]` 항목. 코드 투어의 다음 정거장 |

## 영어/개발 용어

| 용어 | 뜻 |
|---|---|
| **debounce(디바운스)** | "잠깐 기다렸다가 마지막 한 번만 실행". 예: 저장 버튼을 연타해도 0.4초 안의 요청은 뭉쳐서 서버에 한 번만 보냄 |
| **DOM** | 브라우저가 HTML을 읽어 만든 화면의 실체. JS가 `document.getElementById(...)`로 만지는 대상 |
| **factory(팩토리)** | 재료(데이터)를 넣으면 완성품(HTML 문자열)을 찍어내는 함수. `app/ui/components/`의 함수들이 전부 팩토리 |
| **hydrate(하이드레이트)** | 저장돼 있던 데이터를 불러와 앱에 "물을 주듯" 되살리는 것. 새로고침 후에도 데이터가 남아 있는 비결 |
| **lifecycle(라이프사이클)** | 카드가 태어나서(생성) 승인될 때까지 거치는 일생. 그 규칙을 관리하는 파일이 `module-card-lifecycle.service.js` |
| **localStorage(로컬스토리지)** | 브라우저 안의 작은 저장 서랍. 새로고침해도 남지만, 이 브라우저에서만 보임. F12 → Application 탭에서 열람 가능 |
| **persist(퍼시스트)** | "저장하다". 이 코드에서는 localStorage 저장 + 서버 전송 예약을 한꺼번에 뜻함 |
| **render(렌더)** | 데이터를 HTML로 바꿔 화면에 그리는 것. `renderAdminCards` = "관리자 카드 화면을 그려라" |
| **escape(이스케이프)** | 데이터 속의 `<`, `>` 같은 특수문자를 무해한 표기로 바꿔서, 입력된 글자가 코드로 실행되는 사고(XSS)를 막는 것 |
| **route / 라우팅** | 주소창의 `#admin-cards` 같은 해시를 보고 어떤 화면을 보여줄지 결정하는 것. 담당은 `app/router/hash-router.js` |
| **smoke test(스모크 테스트)** | "연기만 나는지 본다" — 앱을 실제 브라우저로 열어 주요 화면이 다 뜨는지 훑는 빠른 검사. `npm run smoke` |
| **vm 검증** | 브라우저 없이 Node.js 안의 가상 공간에서 코드를 실행해보는 검사. `static:validate-*` 명령들이 이 방식 |

## 챕터 01에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **해시(hash)** | 주소 뒤에 붙는 `#부분` (예: `index.html#admin-cards`). 페이지를 새로 받지 않고도 "지금 어느 화면인지"를 주소에 기록하는 방법. 라우터가 이걸 읽고 화면을 고름 |
| **이벤트 리스너(event listener)** | "이 일이 생기면 나를 불러줘"라는 예약. 예: `addEventListener('click', ...)` = 클릭되면 이 함수 실행. 버튼이 동작하는 원리의 전부 |
| **쉘(shell)** | 모든 화면이 공유하는 겉 틀 — 사이드바, 상단바, 알림판. 담당 파일 `app-shell.js`. 액자에 비유 |
| **가드(guard)** | 함수 첫머리의 자격 검사. 조건이 안 맞으면 일을 시작하기 전에 거절함 (403 화면, "QC 완료 후 요청 가능" 에러 등) |
| **403** | "권한 없음"을 뜻하는 관례적 번호. 이 앱에선 권한 없는 화면에 들어가면 보이는 안내 화면 이름 |

## 챕터 02에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **전역 변수(global variable)** | `window.` 으로 시작하는 변수. 코드 어디서든 읽고 쓸 수 있어서 "모든 화면이 공유하는 데이터 통"이 됨. 이 앱에서는 `window.ORDO_MODULE_CARDS` 가 대표적 |
| **배열(Array)** | 여러 개를 순서대로 담는 상자. `[카드1, 카드2, ...]` 형태. `배열[0]`은 첫 번째, `배열.length`는 개수 |
| **객체(Object)** | 이름표가 붙은 칸들의 묶음. `{id:'mc-001', status:'approved', ...}` — 카드 한 장이 객체 하나 |
| **필드(field)** | 객체 안의 칸 하나. 예: 카드 객체의 `status` 필드 = 현재 상태를 담는 칸 |
| **단일 원천(single source of truth)** | "진짜 정보는 이곳 하나에만 있고, 나머지는 여기를 읽는다"는 원칙. 이 앱에서 카드의 단일 원천 = `window.ORDO_MODULE_CARDS` |

## 챕터 03에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **팩토리(factory)** | 재료(데이터)를 넣으면 완성품(HTML 문자열)을 찍어내는 함수. 이 앱의 `app/ui/components/` 함수들이 전부 팩토리. 금형 틀에 플라스틱을 넣으면 모양이 나오는 것과 같음 |
| **tone(색조)** | 카드의 "기분"을 색으로 표현하는 5단계 등급. crit(위험/빨강), warn(경고/노랑), pend(대기/파랑), ok(정상/초록), rej(비활성/회색). 카드 상태와 마감일을 보고 자동 결정됨 |
| **variant(변형)** | 같은 부품이 들어갈 맥락에 따라 모양/크기가 달라지는 것. EmptyState 팩토리가 5가지 variant를 가진 이유 = 대시보드·목록·상세패널의 크기가 제각각이라서 |
| **XSS(크로스 사이트 스크립팅)** | 입력칸에 `<script>` 같은 코드를 넣어 다른 사용자의 브라우저에서 실행시키는 공격. `escapeHtml` 함수가 `<`를 `&lt;`로 바꿔서 방어함 |
| **순수 함수(pure function)** | 같은 입력이면 항상 같은 출력만 돌려주고, 외부 상태를 바꾸지 않는 함수. 팩토리들은 HTML 문자열만 리턴하고 DOM·localStorage를 건드리지 않으므로 순수 함수 |

## 챕터 03 후반부에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **시트(sheet)** | 화면 위에 덮이는 반투명 오버레이 패널. 일괄 생성·편집 같은 부가 작업을 할 때 열림. 모달(modal)과 비슷하지만 화면 일부만 가리는 형태 |
| **옵션 객체(opts)** | 팩토리 함수의 두 번째 인자로 전달하는 설정 묶음. 화면별 차이를 하나의 함수에서 흡수하는 열쇠 |
| **레이아웃(layout)** | 부품의 배치 방식. ActionToolbar의 'wrap'(한 줄 감싸기)과 'stack'(세로→가로 반응형) 같은 것 |
| **aria-hidden** | 스크린 리더(시각 장애인용 보조 기술)에게 "이 요소는 지금 안 보이는 상태"라고 알려주는 HTML 속성 |

## 챕터 04에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **상태 전이(state transition)** | 카드의 상태가 한 단계에서 다음 단계로 바뀌는 것. `pending → in_progress → review → done → approved`. 허용된 경로만 가능하고, 건너뛰기는 가드가 막음 |
| **가드(guard)** | 상태를 바꾸기 전에 "자격이 되는가?" 확인하는 조건 검사. 통과 못 하면 에러를 던져서 변경을 막음. 예: "QC 미완료면 리뷰 요청 불가" |
| **throw(에러 던지기)** | "이 작업은 실패했습니다"를 프로그램에게 알리는 방법. 화면이 try/catch로 잡아서 사용자에게 alert로 보여줌 |
| **감사 로그(audit log)** | "누가 언제 무엇을 했는가"를 남기는 장부. admin-audit 화면에서 열람. 문제 발생 시 추적용 |
| **디바운스(debounce)** | 연타를 뭉쳐서 마지막 한 번만 실행하는 기법. 저장 버튼을 5번 눌러도 0.4초 후 1회만 서버에 보냄 |

## 챕터 05에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **API(에이피아이)** | 프로그램끼리 대화하는 창구. 이 앱에서는 브라우저가 서버에게 "카드 줘(GET)" 또는 "카드 저장해(PUT)"라고 요청하는 주소(`/api/module-cards`)가 API |
| **PUT / GET** | HTTP 메서드. GET = "데이터를 가져와", PUT = "데이터를 넣어/갈아끼워". 브라우저가 서버에게 보내는 "동사" |
| **JSON** | JavaScript Object Notation. `{key: value}` 형태로 데이터를 주고받는 표준 형식. 이 앱의 카드 데이터가 JSON으로 저장/전송됨 |
| **트랜잭션(transaction)** | "전부 성공하거나, 전부 취소하거나" — 중간에 실패하면 절반만 저장되는 사고를 막는 장치. DB에서 카드를 삭제+삽입할 때 사용 |
| **서버리스 함수(serverless function)** | 서버를 직접 관리하지 않고, 요청이 올 때만 잠깐 실행되는 코드. Vercel이 `api/module-cards.js`를 이 방식으로 실행 |

## 챕터 06에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **VM(가상 머신)** | Node.js 안에 만드는 "가짜 브라우저" 환경. 실제 Chrome 없이도 JS 코드를 실행해볼 수 있음. `vm.createContext()`로 생성. 검증 스크립트의 핵심 도구 |
| **assert(어서트)** | "이 조건이 참이어야 한다"는 선언. 거짓이면 에러를 던져서 테스트 실패를 알림. 예: `assert(card.status === 'review')` |
| **headless(헤드리스)** | 브라우저를 화면 없이 실행하는 모드. 사람 눈에는 아무것도 안 보이지만 내부적으로 페이지를 로드하고 DOM을 확인할 수 있음 |
| **CDP(Chrome DevTools Protocol)** | Chrome 브라우저를 프로그램으로 조종하는 통신 규약. smoke-check가 "이 주소로 이동해", "이 요소 있어?" 같은 명령을 보내는 방식 |
| **마커 클래스(marker class)** | 스타일이나 테스트 선택자로 사용되는 필수 CSS 클래스. 실수로 지우면 화면이 깨지거나 테스트가 실패함. 예: `ordo-c-status-badge` |

## 챕터 07 (총정리)에서 추가된 용어

| 용어 | 뜻 |
|---|---|
| **레거시(legacy)** | 이전 방식으로 만들어진 코드. 새 구조가 나왔지만 아직 교체되지 않고 남아있는 것. 예: `workspace.ui.js`의 `moduleCard()` 함수는 `ModuleCardListItem` 팩토리의 레거시 버전 |
| **시드(seed)** | 테스트나 시연을 위해 미리 심어두는 초기 데이터. `seed-module-cards-db.cjs`가 DB에 카드 12장을 심는 역할 |
| **네임스페이스(namespace)** | 이름 충돌을 막기 위한 "이름 공간". `window.ORDO_UI_COMPONENTS`가 35개 팩토리를 하나의 객체 아래 묶어놓는 것이 네임스페이스 패턴 |
