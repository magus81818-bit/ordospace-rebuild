# CODE GUIDE 00 — 시작하기: 이 코드는 어떻게 생겼나

> 이 시리즈는 ORDOSPACE 코드를 **직접 읽고 관리할 수 있게** 만들기 위한 안내서입니다.
> 순서대로 읽고, 각 챕터 끝의 "5분 따라하기"를 꼭 손으로 해보세요.
> 읽기만 한 지식은 증발하지만, 직접 확인한 지식은 남습니다.

---

## 1. 30초 요약 — 이 앱은 무엇인가

ORDOSPACE는 **역할이 3개인 협업 워크스페이스**입니다.

- **관리자(admin)** 가 작업 카드(ModuleCard)를 만들어 작업자에게 배정하고,
- **작업자(worker)** 가 일을 하고 검토를 요청하면,
- 관리자가 확인해서 **클라이언트(client)** 에게 넘기고, 클라이언트가 승인하거나 수정을 요청합니다.

이 모든 과정에서 바뀐 내용은 **브라우저(localStorage)와 서버(Postgres DB)에 저장**되어, 새로고침해도 사라지지 않습니다.

## 2. 전체 지도

브라우저가 앱을 열면 아래 순서로 부품이 조립됩니다. (자세한 이야기는 챕터 01)

```text
 index.html  ─ 모든 화면의 뼈대(HTML)와 스크립트 로드 순서표
     │
     ├─ app/config/      "설정표" ─ 역할·화면 목록·메뉴 등 앱 전체가 공유하는 상수
     ├─ app/data/        "창고"   ─ 시연용 샘플 데이터 (카드 12장, 사람들, 프로젝트)
     ├─ app/services/    "심판"   ─ 카드 상태를 바꾸는 규칙 + 저장 담당 (lifecycle 서비스)
     ├─ app/ui/components/ "부품 공장" ─ 배지·카드·목록 같은 HTML 조각을 찍어내는 함수들
     ├─ app/router/      "교통정리" ─ 주소(#admin-cards 같은 해시)를 보고 어떤 화면을 보여줄지 결정
     ├─ app/layout/      "액자"   ─ 사이드바·상단바·알림판 등 모든 화면이 공유하는 틀
     ├─ app/screens/     "무대"   ─ 역할별 화면. 데이터를 골라 부품 공장에 주문하고, 버튼에 반응
     └─ app/qa/          "검사원" ─ 화면이 제대로 뜨는지 자동으로 눌러보는 테스트

 (서버 쪽 — 브라우저 밖에서 실행)
 api/module-cards.js     "우체국" ─ 카드 전체를 받아 DB에 저장(PUT)하거나 꺼내주는(GET) 창구
 api/_lib/…repository    "금고지기" ─ 실제 Postgres DB 읽기/쓰기와 데이터 검사
 tools/validate-*.cjs    "감리단" ─ 코드를 고친 뒤 돌려보는 자동 검증 스크립트들
```

**데이터가 흐르는 길** (가장 중요한 그림 — 챕터 04·05에서 자세히):

```text
 [버튼 클릭] → screens/ 가 감지 → services/lifecycle 이 규칙 확인 후 상태 변경
     → ① localStorage에 즉시 저장 (새로고침 대비)
     → ② 0.4초 뒤 서버(/api/module-cards)로 전체 전송 → Postgres DB
     → 화면 다시 그리기 (부품 공장에서 새 HTML을 받아 갈아끼움)
```

## 4. 코드 주석 읽는 법

모든 JS 파일 맨 위에는 아래 형식의 **파일 헤더**가 있습니다 (주석화 라운드가 진행되면서 채워집니다):

```js
/* ============================================================
   [이 파일은]      한 줄 요약 — 이 파일의 정체
   [언제 실행]      페이지 로드 시 / 버튼 클릭 시 / 서버에서 등
   [주요 등장인물]  이 파일의 핵심 함수·객체 목록
   [연결]          ← 이 파일을 사용하는 곳 / → 이 파일이 의존하는 곳
   [다음 읽을 파일] 코드 투어의 다음 정거장
   [수정할 때 주의] 건드리기 전에 알아야 할 함정
   ============================================================ */
```

- **파일을 처음 열면 헤더만 읽어도 됩니다.** 그 파일의 정체를 아는 게 목적입니다.
- 본문의 `/* ── 섹션명 ── */` 는 파일 안의 큰 덩어리 구분입니다.
- 함수 위 주석은 "목적 / 입력 / 반환 / **부수효과**"를 알려줍니다. 부수효과(저장이 되는지, 화면이 다시 그려지는지)가 가장 중요합니다.
- `[다음 읽을 파일]` 이정표만 따라가면 문서 없이 코드만으로 투어가 이어집니다.

## 5. 5분 따라하기 — 첫 실습

1. 프로젝트 폴더의 `index.html`을 브라우저(Chrome)로 엽니다.
2. `F12` 를 눌러 개발자 도구를 열고 **Console** 탭으로 갑니다.
3. 아래를 입력하고 Enter:
   ```js
   window.ORDO_MODULE_CARDS.length
   ```
   → `12` 가 나오면, 창고(app/data)에 카드 12장이 실려 있다는 뜻입니다.
4. 하나 더:
   ```js
   window.ORDO_MODULE_CARDS[0].module
   ```
   → 첫 번째 카드의 제목이 나옵니다. **화면에 보이는 모든 카드가 결국 이 배열에서 나옵니다.**
